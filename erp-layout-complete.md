```
右侧悬浮框顶部：
┌─────────────────────────────────────────────────────────┐
```

### 7.3 选择模式规则

1. **点击"选择"按钮** → 进入选择模式
2. **订单项左侧出现复选框** → 可勾选/取消
3. **支持拖拽排序** → 调整开票顺序
4. **右侧显示已选项** → 实时统计
5. **点击空白处或"完成"** → 退出选择模式

### 7.4 跨订单选择流程

```
步骤 1：在订单列表中搜索客户名称
        ↓
步骤 2：显示该客户的所有订单
        ↓
步骤 3：点击订单 A 详情 → 点击"选择" → 勾选订单项
        ↓
步骤 4：点击空白处返回订单列表
        ↓
步骤 5：点击订单 B 详情 → 点击"选择" → 继续勾选
        ↓
步骤 6：重复步骤 4-5，直到选完所有需要的订单项
        ↓
步骤 7：在右侧选择框中点击"联合开票"或"开票"
        ↓
步骤 8：弹出确认窗口 → 自定义开票内容 → 确认
        ↓
步骤 9：完成开票 → 返回订单列表
```

### 7.5 联合开票 vs 普通开票

```
条件判断：
- 所有已选项来自同一订单 → 显示"开票"按钮
- 已选项来自多个订单 → 显示"联合开票"按钮
- 两种按钮互斥，不同时显示

联合开票限制：
- 必须是同一客户的不同订单
- 通过搜索客户名称来筛选订单
```

## 十三、总结

### 核心设计原则

1. **单页操作**：所有操作在列表页完成，无需跳转
2. **数据传递**：通知直接关联业务数据，点击即可操作
3. **功能散落**：按钮放在卡片上，直观可见
4. **权限隐藏**：无权限的按钮直接不显示
5. **C 端体验**：类似淘宝订单、滴滴行程

### 布局优势

| 特点 | 说明 |
|------|------|
| 左侧卡片列表 | 信息密度高，可滚动 |
| 右侧悬浮框 | 操作入口固定，始终可见 |
| 抽屉左右分栏 | 订单项左 60%，详情右 40% |
| 通知即操作 | 点击通知直接处理业务 |
| 选择模式 | 支持跨订单多选/拖拽 |
| 面包屑导航 | 清晰的位置感知 |
